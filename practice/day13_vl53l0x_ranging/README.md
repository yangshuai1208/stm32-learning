# Day13 VL53L0X 测距实验

## 一、实验目标
在 STM32F103 平台上移植并调通 VL53L0X 激光测距模块，实现基础测距数据读取，并通过串口输出结果。

## 二、实验内容
本次实验完成了以下内容：

- 配置 I2C 外设用于与 VL53L0X 通信
- 移植 VL53L0X 官方 API
- 编写 platform 层 I2C 读写适配函数
- 编写应用层测距逻辑
- 通过串口输出测距结果
- 使用板级驱动模块组织代码结构

## 三、工程结构
本实验主要分为以下几个部分：

- `Api/`：VL53L0X 官方 API
- `platform/`：VL53L0X 平台适配层
- `app_vl53l0x.c`：应用层测距逻辑
- `bsp_i2c.c`：I2C 底层驱动
- `bsp_usart.c`：串口初始化与调试输出
- `bsp_dwt.c`：精确延时
- `bsp_led.c`：LED 指示灯驱动

## 四、硬件连接
### 1. VL53L0X
- SDA：接 STM32 I2C SDA
- SCL：接 STM32 I2C SCL
- VCC：3.3V
- GND：GND

### 2. 串口
- TX：接串口模块 RX
- RX：接串口模块 TX
- 波特率：115200

## 五、程序流程
1. 初始化 HAL 库和系统时钟
2. 初始化串口
3. 初始化 I2C
4. 初始化 VL53L0X
5. 进入主循环读取距离数据
6. 通过串口输出测距结果

## 六、关键知识点
通过本实验，主要学习了以下内容：

- 传感器官方 API 移植方法
- platform 适配层的作用
- I2C 多字节读写接口设计
- STM32 与 VL53L0X 通信流程
- 应用层与底层驱动分层组织方式

## 七、遇到的问题
本次实验中遇到的主要问题：

- `vl53l0x_i2c_rw.h` 与 `vl53l0x_i2c_platform.h` 中函数声明不一致
- 平台层返回值类型和参数类型冲突
- 部分头文件引用路径仍是旧工程风格
- `Error_Handler` 声明不完整导致 warning

## 八、解决方法
- 统一 `vl53l0x_i2c_rw.h` 和 `vl53l0x_i2c_rw.c` 的函数签名
- 以 `vl53l0x_i2c_platform.h` 的接口定义为准
- 统一头文件包含路径
- 完善板级驱动头文件引用

## 九、实验现象
下载程序后，串口能够输出 VL53L0X 读取到的距离数据，表明 I2C 通信与传感器驱动移植基本成功。

## 十、实验总结
本实验完成了 VL53L0X 激光测距模块在 STM32F103 平台上的基础移植和联调，为后续环境感知、避障、距离检测类项目打下基础。

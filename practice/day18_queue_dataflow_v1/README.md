# Day18 队列数据流打通 v1

## 一、项目目标
在前面已经完成 FreeRTOS + DHT11 + OLED + 串口输出三任务结构的基础上，进一步引入 FreeRTOS 队列机制，实现“共享状态 + 消息队列”混合架构，让系统的数据流更清晰。

## 二、今天完成的内容
今天主要完成了以下工作：

- 保留原有三任务结构
- 继续使用共享结构体保存系统当前状态
- 新增 `SensorMessage` 消息结构体
- 新增传感器消息队列
- 采集任务在每次采样后发送最新消息
- 串口任务改为从队列接收消息后输出
- OLED 任务继续读取共享结构体进行显示

## 三、工程结构说明

### 1. main.c
负责系统整体初始化，包括：

- HAL 初始化
- GPIO 初始化
- 系统时钟配置
- USART1 初始化
- I2C1 初始化
- DWT 初始化
- DHT11 数据引脚初始化
- OLED 初始化
- FreeRTOS 初始化
- 启动调度器

### 2. freertos.c
当前系统包含 3 个核心任务：

- `sensorTask`：负责采集 DHT11 温湿度
- `displayTask`：负责 OLED 显示
- `uartTask`：负责串口输出

同时包含两类数据组织方式：

- `SensorData`：共享结构体，保存当前系统状态
- `SensorMessage`：队列消息结构体，用于串口任务接收最新采样结果

## 四、当前任务架构

### 1. sensorTask
功能：
- 周期读取 DHT11 温湿度
- 更新共享结构体 `g_sensor_data`
- 成功时增加 `update_count`
- 失败时增加 `error_count`
- 每次采样后构造 `SensorMessage`
- 使用 `xQueueOverwrite()` 将最新消息写入队列

周期：
- 2000ms

### 2. displayTask
功能：
- 从共享结构体中读取快照
- OLED 显示当前状态

显示逻辑：
- 未拿到第一组数据：显示 `WAIT DATA`
- 当前数据有效：显示温湿度和 `CNT`
- 当前数据无效：显示 `READ ERROR` 和 `ERR`

周期：
- 1000ms

### 3. uartTask
功能：
- 从消息队列中阻塞接收 `SensorMessage`
- 收到新消息后，通过串口输出当前状态
- 翻转 LED 和板载灯作为运行心跳

特点：
- 不再定时轮询共享结构体
- 而是“有新消息就输出一次”

## 五、为什么今天采用“共享结构体 + 队列”混合架构

今天没有把所有数据都强行改成队列，而是做了职责划分：

### 1. 共享结构体适合保存“当前状态”
OLED 任务需要随时显示系统当前最新状态，因此继续读取共享结构体最直观。

### 2. 队列适合传递“新采样事件”
串口任务更适合在“每次有新数据到来时”输出一次，因此改成通过队列接收消息更合理。

### 3. 队列长度设置为 1
当前队列长度为 1，配合 `xQueueOverwrite()` 使用，表示：

- 不保存历史旧消息
- 永远只保留最新一条采样结果

这种方式很适合当前温湿度采样系统。

## 六、共享结构体与消息结构体

### 1. SensorData
包含：

- 温度整数部分
- 温度小数部分
- 湿度整数部分
- 湿度小数部分
- `ready`
- `valid`
- `update_count`
- `error_count`

用途：
- 保存系统当前最新状态
- 供 OLED 任务显示使用

### 2. SensorMessage
包含：

- 温度整数部分
- 温度小数部分
- 湿度整数部分
- 湿度小数部分
- `valid`
- `update_count`
- `error_count`

用途：
- 作为“采样事件消息”
- 通过队列发送给串口任务

## 七、硬件连接

### OLED
- SCL -> PB6
- SDA -> PB7
- VCC -> 3.3V
- GND -> GND

### DHT11
- DATA -> PB12
- VCC -> 3.3V 或 5V
- GND -> GND

### 串口
- USART1_TX -> PA9
- USART1_RX -> PA10
- 波特率 -> 115200

## 八、实验现象

### 1. 刚启动时
如果 OLED 初始化成功，OLED 会先显示：

- `DHT11`
- `WAIT DATA`

表示系统已经启动，但第一组温湿度还未采集到。

### 2. 采集成功时
OLED 显示：

- `DHT11`
- `TEMP:xx.xC`
- `HUMI:xx.x%`
- `CNT:xx`

串口输出类似：

- `DHT11 TEMP:25.0C HUMI:60.0% CNT:15 ERR:1`

### 3. 采集失败时
OLED 显示：

- `DHT11`
- `READ ERROR`
- `ERR:xx`

串口输出类似：

- `DHT11 READ ERROR CNT:15 ERR:3`

### 4. LED 现象
串口任务每次收到新消息后，翻转红灯和板载灯，表示队列链路和任务调度正常工作。

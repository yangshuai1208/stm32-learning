## 三、工程结构说明
### 1. main.c
负责完成系统整体初始化，包括：

- HAL 初始化
- 系统时钟配置
- GPIO 初始化
- USART1 初始化
- I2C1 初始化
- DWT 初始化
- DHT11 引脚初始化
- OLED 初始化
- FreeRTOS 初始化
- 启动调度器

### 2. freertos.c
负责创建和管理任务，当前工程主要包含以下任务：

- `sensorTask`：采集 DHT11 温湿度数据
- `ledTask`：负责 OLED 显示
- `uartTask`：负责串口输出和 LED 心跳
- `defaultTask`：当前仅保留最基本空转任务，后续可删除优化

### 3. DHT11 驱动
由以下文件实现：

- `bsp_dht11.c`
- `bsp_dht11.h`

实现功能：
- DHT11 数据引脚初始化
- DHT11 单总线时序读取
- 温湿度数据校验

### 4. OLED 驱动
由以下文件实现：

- `bsp_i2c_oled.c`
- `bsp_i2c_oled.h`
- `bsp_fonts.c`
- `bsp_fonts.h`

实现功能：
- OLED 初始化
- I2C 写命令与写数据
- OLED 字符串显示

### 5. DWT 延时模块
由以下文件实现：

- `bsp_dwt.c`
- `bsp_dwt.h`

实现功能：
- 微秒级延时
- 为 DHT11 时序读取提供精确延时基础

## 四、任务设计
### 1. sensorTask
功能：
- 调用 DHT11 驱动读取温湿度
- 更新共享数据结构体
- 标记当前数据是否有效

特点：
- 只负责采集
- 不直接显示
- 不直接串口打印

### 2. ledTask
功能：
- 从共享数据结构体读取温湿度
- 使用 OLED 显示温湿度内容
- 若采集失败，则显示错误提示

特点：
- 只负责显示
- 不直接读 DHT11

### 3. uartTask
功能：
- 从共享数据结构体读取温湿度
- 串口输出当前温湿度状态
- 翻转 LED 作为系统运行心跳

特点：
- 只负责输出
- 便于调试与观察系统状态

## 五、硬件连接
### OLED
- SCL -> PB6
- SDA -> PB7
- VCC -> 3.3V
- GND -> GND

### DHT11
- DATA -> PB12
- VCC -> 3.3V 或 5V
- GND -> GND

### 串口
- USART1_TX -> PA9
- USART1_RX -> PA10
- 波特率 -> 115200

## 六、实验现象
### 1. 启动现象
上电后串口会输出启动信息，例如：

- `BOOT`
- `OLED READY` 或 `OLED FAIL`

### 2. OLED 显示现象
正常情况下，OLED 周期显示：

- `DHT11`
- `TEMP:xx.xC`
- `HUMI:xx.x%`

如果采集失败，则显示：

- `DHT11`
- `READ ERROR`

### 3. 串口现象
正常情况下，串口周期输出：

- `TEMP:xx.xC HUMI:xx.x%`

如果采集失败，则输出：

- `DHT11 READ ERROR`

### 4. LED 现象
LED 周期翻转，说明系统仍在正常运行。

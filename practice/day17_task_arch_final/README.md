# Day17 任务架构最终版

## 一、项目目标
在前面已经完成 FreeRTOS + DHT11 + OLED + 串口输出的基础上，进一步整理任务结构，形成一个职责清晰、状态可观测、便于后续扩展的多任务温湿度监测系统。

## 二、今天完成的内容
今天主要完成了以下工作：

- 固定为 3 个核心任务架构
- 明确采集、显示、输出三类职责
- 继续使用共享结构体传递温湿度数据
- 增加系统状态字段
- OLED 显示逻辑定型
- 串口输出逻辑定型
- 保留系统错误处理和状态指示机制

## 三、工程结构说明

### 1. main.c
负责系统整体初始化，包括：

- HAL 初始化
- GPIO 初始化
- 系统时钟配置
- USART1 初始化
- I2C1 初始化
- DWT 初始化
- DHT11 数据引脚初始化
- OLED 初始化
- FreeRTOS 初始化
- 启动调度器

### 2. freertos.c
当前系统包含 3 个核心任务：

- `sensorTask`：负责采集 DHT11 温湿度
- `displayTask`：负责 OLED 显示
- `uartTask`：负责串口输出

### 3. 共享数据结构体
当前共享结构体 `SensorData` 包含以下字段：

- `temp_int`：温度整数部分
- `temp_dec`：温度小数部分
- `humi_int`：湿度整数部分
- `humi_dec`：湿度小数部分
- `ready`：是否已经获取过采样结果
- `valid`：当前采样是否有效
- `update_count`：成功采集次数
- `error_count`：失败次数

## 四、任务设计

### 1. sensorTask
功能：
- 调用 `DHT11_ReadData()` 周期读取温湿度
- 更新共享结构体中的温湿度数据
- 成功时 `update_count++`
- 失败时 `error_count++`
- 更新 `ready` 和 `valid` 状态位

周期：
- 2000ms

### 2. displayTask
功能：
- 从共享结构体读取当前温湿度状态
- OLED 显示当前系统状态

显示逻辑：
- 未获取到第一组数据：显示 `WAIT DATA`
- 当前数据有效：显示温湿度和采集次数
- 当前数据无效：显示 `READ ERROR` 和失败次数

周期：
- 1000ms

### 3. uartTask
功能：
- 从共享结构体读取当前温湿度状态
- 串口输出当前温湿度或错误状态
- 翻转 LED 作为系统运行心跳

周期：
- 2000ms

## 五、硬件连接

### OLED
- SCL -> PB6
- SDA -> PB7
- VCC -> 3.3V
- GND -> GND

### DHT11
- DATA -> PB12
- VCC -> 3.3V 或 5V
- GND -> GND

### 串口
- USART1_TX -> PA9
- USART1_RX -> PA10
- 波特率 -> 115200

## 六、实验现象

### 1. 刚启动时
如果 OLED 初始化成功，屏幕先显示：

- `DHT11`
- `WAIT DATA`

表示系统已经启动，但还没有拿到第一组温湿度数据。

### 2. 采集成功时
OLED 显示：

- `DHT11`
- `TEMP:xx.xC`
- `HUMI:xx.x%`
- `CNT:xx`

串口输出类似：

- `DHT11 TEMP:25.0C HUMI:60.0% CNT:15`

### 3. 采集失败时
OLED 显示：

- `DHT11`
- `READ ERROR`
- `ERR:xx`

串口输出类似：

- `DHT11 READ ERROR CNT:15 ERR:3`

### 4. LED 现象
红灯和板载 LED 周期翻转，表示系统任务仍在运行。

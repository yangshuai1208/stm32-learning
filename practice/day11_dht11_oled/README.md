# Day12 DHT11 + OLED 温湿度显示实验

## 一、实验目标
使用 STM32F103 读取 DHT11 温湿度传感器数据，并通过 I2C OLED 屏幕显示温度和湿度信息。

## 二、实验内容
本次实验完成了以下功能：

- 初始化 USART1 串口
- 初始化 DWT 延时模块
- 初始化 I2C1 总线
- 初始化 OLED 显示屏
- 初始化 DHT11 数据引脚
- 周期性读取 DHT11 温湿度数据
- 在 OLED 上显示温度和湿度信息

## 三、模块划分
本实验采用模块化方式组织代码：

- `bsp_dht11`：DHT11 底层时序读取
- `app_dht11_oled`：温湿度读取与 OLED 显示逻辑
- `bsp_dwt`：微秒/毫秒延时
- `bsp_i2c`：I2C1 初始化
- `bsp_i2c_oled`：OLED 驱动与显示函数
- `bsp_fonts`：字库
- `bsp_usart`：串口初始化与 printf 重定向

## 四、硬件连接
### 1. DHT11
- DATA：PB12
- VCC：3.3V 或 5V
- GND：GND

### 2. OLED(I2C)
- SCL：PB6
- SDA：PB7
- VCC：3.3V
- GND：GND

### 3. 串口
- USART1_TX：PA9
- USART1_RX：PA10
- 波特率：115200

## 五、程序流程
1. 初始化串口
2. 初始化 DWT 精确延时
3. 初始化 I2C1 和 OLED
4. 初始化 DHT11 数据引脚
5. 主循环中调用 `Dht11_ReadAndShow()`
6. 读取成功时在 OLED 上显示温度和湿度
7. 读取失败时在 OLED 上显示错误提示

## 六、关键知识点
通过本实验，主要学习了以下内容：

- DHT11 单总线时序读取
- GPIO 动态切换输入/输出模式
- DWT 精确延时的使用
- I2C OLED 屏幕初始化与显示
- 模块化工程组织方式
- 温湿度数据显示逻辑

## 七、实验现象
下载程序后，OLED 屏幕能够显示固定标题以及 DHT11 采集到的温度、湿度数据；若读取失败，则显示错误提示。

## 八、当前问题与改进
- DHT11 读取时序对延时精度要求较高
- 若传感器接线错误或响应异常，可能读取失败
- 后续可以增加串口调试输出，便于定位读取失败原因

## 九、实验总结
本实验完成了 DHT11 与 OLED 的基本联调，实现了温湿度数据采集与屏幕显示。这是后续环境监测系统中“传感器采集 + 显示输出”功能的重要基础。

# Day25 FreeRTOS 互斥量保护共享资源 v1

## 一、项目目标

本项目基于前一天完成的按键修改报警阈值和 Flash 保存功能，继续引入 FreeRTOS 互斥量机制，用于保护多任务系统中的共享资源。

第25天的重点是：

- 使用互斥量保护串口输出
- 使用互斥量保护 OLED 显示
- 使用互斥量保护配置结构体 `g_app_config`
- 避免多个任务同时访问同一个资源造成冲突
- 提升 FreeRTOS 项目的工程规范性

本日不是新增外设，而是对已有系统进行多任务资源访问规范化。

---

## 二、当前系统功能

当前系统已经具备：

- DHT11 温湿度采集
- OLED 实时显示
- 串口状态输出
- 温湿度阈值报警判断
- 按键修改报警阈值
- Flash 保存配置参数
- IWDG 看门狗任务健康检测
- FreeRTOS 多任务调度
- 队列传递传感器消息
- 互斥量保护共享资源

---

## 三、当前任务架构

当前系统包含以下任务：

### 1. sensorTask

负责：

- 读取 DHT11 温湿度数据
- 获取配置参数快照
- 根据温湿度和配置阈值判断报警状态
- 更新共享结构体 `g_sensor_data`
- 构造 `SensorMessage`
- 使用队列发送给串口任务

### 2. displayTask

负责：

- 从 `g_sensor_data` 读取当前系统状态
- 使用 OLED 显示温度、湿度和报警状态
- 使用 OLED 互斥量保护 OLED / I2C 显示资源

### 3. uartTask

负责：

- 从队列接收传感器消息
- 获取配置参数快照
- 串口输出温湿度、报警状态、成功次数、失败次数和当前阈值
- 使用串口互斥量保护 USART1 输出

### 4. keyTask

负责：

- 扫描 MODE / UP / DOWN 按键
- 修改温度报警上限或湿度报警下限
- 使用配置互斥量保护 `g_app_config`
- 修改后保存配置到 Flash

### 5. watchdogTask

负责：

- 检查 sensorTask、displayTask、uartTask 是否持续运行
- 三个关键任务都正常时才喂 IWDG

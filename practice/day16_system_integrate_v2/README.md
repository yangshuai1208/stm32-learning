# Day16 状态监测与异常统计 v2

## 一、项目目标
在前一天完成 FreeRTOS + DHT11 + OLED + 串口输出的基础上，继续对工程进行升级，使系统不仅能显示温湿度，还能记录采集状态、统计成功次数和失败次数，提升系统可观测性和稳定性。

## 二、今天完成的内容
今天主要完成了以下内容：

- 删除无实际业务意义的默认任务
- 保留 3 个核心任务：采集、显示、输出
- 增加共享数据状态字段
- 增加成功采集次数统计
- 增加失败次数统计
- OLED 增加状态显示
- 串口增加状态输出
- 继续保留 LED 作为系统运行心跳

## 三、工程结构说明

### 1. main.c
负责完成系统整体初始化，包括：

- HAL 初始化
- GPIO 初始化
- 系统时钟配置
- USART1 初始化
- I2C1 初始化
- DWT 初始化
- DHT11 数据引脚初始化
- OLED 初始化
- FreeRTOS 初始化
- 启动调度器

### 2. freertos.c
当前工程主要包含 3 个任务：

- `sensorTask`：负责采集 DHT11 温湿度
- `ledTask`：负责 OLED 显示
- `uartTask`：负责串口输出

同时，通过共享结构体 `SensorData` 保存当前状态数据。

### 3. SensorData 结构体
当前共享数据结构体包含：

- 温度整数部分
- 温度小数部分
- 湿度整数部分
- 湿度小数部分
- `ready`：是否已经获取过采样结果
- `valid`：当前采样是否有效
- `update_count`：成功采集次数
- `error_count`：失败次数

### 4. DHT11 驱动
由以下文件实现：

- `bsp_dht11.c`
- `bsp_dht11.h`

主要功能：

- GPIO 配置
- 单总线时序控制
- 超时等待
- 温湿度数据读取
- 校验和判断

### 5. OLED 驱动
由以下文件实现：

- `bsp_i2c_oled.c`
- `bsp_i2c_oled.h`

主要功能：

- OLED 设备检测
- OLED 初始化
- 清屏
- 字符串显示
- I2C 通信显示输出

## 四、任务设计

### 1. sensorTask
功能：
- 调用 `DHT11_ReadData()` 周期读取温湿度
- 成功时更新共享数据
- 成功时 `update_count++`
- 失败时 `error_count++`
- 更新 `ready` 和 `valid` 状态位

周期：
- 2000ms

### 2. ledTask
功能：
- 从共享结构体读取当前温湿度和状态
- OLED 显示当前数据
- 当数据未准备好时显示 `WAIT DATA`
- 当采集失败时显示 `READ ERROR`
- 显示采集次数或错误次数

周期：
- 1000ms

### 3. uartTask
功能：
- 从共享结构体读取当前温湿度和状态
- 串口输出当前温湿度
- 输出成功次数和失败次数
- 用于后续稳定性测试和调试观察

周期：
- 2000ms

## 五、硬件连接

### 1. OLED
- SCL -> PB6
- SDA -> PB7
- VCC -> 3.3V
- GND -> GND

### 2. DHT11
- DATA -> PB12
- VCC -> 3.3V 或 5V
- GND -> GND

### 3. 串口
- USART1_TX -> PA9
- USART1_RX -> PA10
- 波特率 -> 115200

## 六、实验现象

### 1. 上电后
如果 OLED 初始化成功，OLED 会先显示：

- `DHT11`
- `WAIT DATA`

表示系统已经启动，但还未拿到第一组传感器数据。

### 2. 正常采集时
OLED 显示：

- `DHT11`
- `TEMP:xx.xC`
- `HUMI:xx.x%`
- `CNT:xx`

串口输出类似：

- `DHT11 TEMP:25.0C HUMI:60.0% CNT:15`

### 3. 采集失败时
OLED 显示：

- `DHT11`
- `READ ERROR`
- `ERR:xx`

串口输出类似：

- `DHT11 READ ERROR CNT:15 ERR:3`

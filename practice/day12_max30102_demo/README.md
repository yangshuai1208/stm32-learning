# MAX30102 心率血氧传感器（STM32，I2C，FIR滤波）

## 功能
- 通过 I2C 读取 MAX30102 传感器的红外（IR）和红光（RED）原始数据
- 使用 ARM CMSIS-DSP 库的 FIR 滤波器对数据进行低通滤波
- 根据滤波后的波形计算心率（次/分钟）和血氧饱和度（SpO2，百分比）
- 通过串口打印结果，同时用 LED 指示数据有效性
- 支持中断方式（FIFO 满中断），提高实时性

## 硬件连接
- MCU：STM32F103C8T6（野火开发板）
- MAX30102 模块：
  - VCC → 3.3V
  - GND → GND
  - SCL → PB6（I2C2）
  - SDA → PB7（I2C2）
  - INT → PB8（中断引脚，可选）
- 串口：USART1（PA9 TX, PA10 RX），115200
- 状态 LED：绿色 LED 指示有效数据采集，红色 LED 用于调试（中断触发时翻转）

## 使用方法
1. 用 Keil 5 打开工程，确保已添加 CMSIS-DSP 库（arm_math.h）
2. 编译下载到开发板
3. 打开串口助手（115200, 8N1）
4. 将手指轻贴在传感器上，等待绿色 LED 闪烁
5. 串口会打印类似：`心率：78 次/min   血氧：97.23 %`

## 代码结构
- `bsp_max30102.c/h`：底层 I2C 读写、寄存器配置、中断初始化、FIFO 读取
- `max30102_fir.c/h`：FIR 滤波器初始化与滤波函数（使用 CMSIS-DSP）
- `app_max30102.c/h`：数据处理任务，缓存有效数据，计算心率和血氧
- `main.c`：初始化外设，循环调用 `MAX30102_Task()`

## 关键参数
- 采样率：200Hz（配置为 200Hz，平均 4 次，实际有效采样率 50Hz）
- LED 电流：约 15mA（寄存器 0x2F）
- 数据有效性阈值：`PPG_DATA_THRESHOLD = 100000`
- 每次计算所需数据量：`TEST_NUMS = 150`（约 3 秒数据）

## 注意事项
- 手指必须完全覆盖传感器，并保持静止，否则数据波动大
- 心率算法基于峰值检测，血氧算法使用 Maxim 官方经验公式
- 必须包含 CMSIS-DSP 库，否则 FIR 滤波无法编译
- 如果串口一直打印“读取失败”，请检查 I2C 连接和模块供电
